3 Architectural Approaches

Smart Video Loop — YouTube Chrome Extension

Recap: What we're building

Scope confirmed

  • Chrome extension, YouTube only
  • Set Start/End via buttons + draggable timeline markers
  • Continuous A→B→A→B looping
  • Save loop points per video (local storage)
A

Content Script Only

Đơn giản nhất. Một content script duy nhất inject vào youtube.com, trực tiếp thao tác với YouTube player API (seek, getCurrentTime, setLoop). UI được tạo bằng DOM injection vào player controls.

Pros

  • Code ít, triển khai nhanh
  • Không cần background service worker
  • Trực tiếp tương tác YouTube player → ít delay

Cons

  • Khó scale nếu sau này thêm platform khác
  • Storage logic nằm chung → khó test riêng
  • Ít tách biệt giữa UI và logic loop
B

Split Architecture (Recommended)

Content Script + Service Worker. Content script lo UI + tương tác DOM/player. Service worker lo persistence (chrome.storage.local) và coordination. Giao tiếp qua chrome.runtime messaging.

Pros

  • Tách biệt rõ: UI ↔ Logic ↔ Storage
  • Dễ test từng phần riêng biệt
  • Service worker có thể xử lý thêm tính năng sau (keyboard shortcuts, notifications)
  • Theo đúng best practices của Chrome Extension Manifest V3

Cons

  • Cần boilerplate messaging giữa content script và SW
  • Phức tạp hơn một chút so với approach A
C

Micro-Modules Architecture

Content Script + Service Worker + nhiều module nhỏ. Chia nhỏ thành các module: loop-engine, timeline-ui, storage-manager, youtube-detector. Mỗi module có interface rõ ràng.

Pros

  • Codebase rất sạch, dễ maintain
  • Dễ mở rộng thêm platform hoặc engine khác
  • Unit test dễ dàng cho từng module

Cons

  • Over-engineering cho MVP
  • Build step có thể cần (bundler)
  • Thời gian phát triển ban đầu lâu hơn

Recommendation: B — đủ sạch để maintain, đủ đơn giản để ship nhanh, dễ upgrade lên C nếu cần sau này.
