Module Details — Part 2: Storage + Messaging

Plus: state machine, error handling, and edge cases

Module 4: Storage Manager

Interface:

saveLoop(videoId, start, end) → Promise<void>
loadLoop(videoId) → Promise<{start, end} | null>
deleteLoop(videoId) → Promise<void>
getAllLoops() → Promise<Record<string, {start,end}>>

Storage Schema (chrome.storage.local):

{
  "loops": {
    "dQw4w9WgXcQ": { start: 90.5, end: 225.3, savedAt: 1706000000 },
    "9bZkp7q19f0": { start: 30.0, end: 60.0, savedAt: 1706000500 }
  }
}

Design decisions:

  • Lưu qua Service Worker → tập trung logic, dễ debug
  • Key theo video ID → lookup O(1)
  • savedAt timestamp → cleanup policy sau này nếu cần (VD: xóa settings cũ hơn 6 tháng)
  • Auto-save sau mỗi lần set start/end → không cần nút Save thủ công
Module 5: Message Bus

Message Protocol:

// Content Script → Service Worker

{ type: "LOOP_SAVE", payload: { videoId, start, end } }
{ type: "LOOP_LOAD", payload: { videoId } }
{ type: "LOOP_DELETE", payload: { videoId } }

// Service Worker → Content Script

{ type: "LOOP_LOADED", payload: { videoId, loop: {start,end} | null } }
{ type: "LOOP_SAVED", payload: { videoId, ok: boolean } }

// Content Script → Content Script (broadcast across tabs)

{ type: "KEYBOARD_SHORTCUT", payload: { action: "toggle_loop"|"set_start"|"set_end" } }

Error handling:

  • Timeout 3s cho message → fallback gracefully
  • SW bị terminated (idle) → re-send message, SW tự restart
  • Network error → không ảnh hưởng (local-only extension)

Loop State Machine

IDLE → READY (có start+end) → LOOPING → PAUSED (ad/user-seek)
IDLE: no loop points set
READY: both A and B set, not yet looping
LOOPING: actively looping A→B
PAUSED: loop temporarily suspended (ad, user manual seek)
Transitions triggered by: user clicks, loop engine detects out-of-range, ad events
