Data Flow: Looping in Action

1. YouTube Detector → detects watch page, signals modules to init
2. Storage Manager → loads saved loop points for this video ID
3. Timeline UI → injects Start/End buttons + markers into player
4. User → clicks "Set Start" at 1:30, then "Set End" at 3:45
5. Loop Engine → starts monitoring: if currentTime ≥ 3:45 → seek to 1:30
6. Storage Manager → saves {start: 90, end: 225} via Message Bus → SW
7. Loop Engine → continues A→B→A→B until user disables or closes page

Key Design Decisions

Why Loop Engine uses requestAnimationFrame

  • Kiểm tra currentTime mỗi frame (~16ms) → phản ứng nhanh, mượt
  • setInterval không đủ chính xác cho seek chính xác
  • Nhẹ hơn MutationObserver

Why ESBuild for bundling

  • TypeScript support, tree shaking, cực nhanh
  • Output: 1 bundle cho content script, 1 cho service worker
  • Giữ module structure trong source nhưng optimize cho production

Mỗi module có trách nhiệm duy nhất, có thể test unit độc lập, và giao tiếp qua interface rõ ràng.
